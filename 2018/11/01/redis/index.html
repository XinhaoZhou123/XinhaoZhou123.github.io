Different from relational database like mysql or oracle, redis is a key-value database. It is a in-memory database so that the speed of accessing redis is really fast. At the same time, redis supports data persistence. When reloading it, redis can store the data to the disk. Redis supports different types of data storage, for instance, string, list, hash ,set, zset.

Remote connect redis

  • Modify IP binding of the configuration file.

    bind 0.0.0.0 OR #bind 127.0.0.1

  • Modify the protected mode of the configuration file.

    protected-mode no

  • Use this command to connect the redis with IP and port number.

    redis-cli -h host -p port -a password

Connect redis with java

`public static void testping(){
    Jedis jedis = new Jedis("10.25.39.152",6379);
    jedis.auth("yourpassword");
    System.out.println(jedis.ping());

    jedis.close();
}`

Test whether the remote connection of redis is successful using this codes. If the response is “pong”, then the connection is set up.

Redis also provides the transaction function. 

`public static void testTransaction(){
    Jedis jedis = new Jedis("10.25.19.145",6379);
    jedis.auth("yourpassword");
    Transaction transaction = jedis.multi();
    //your implementation
    transaction.set("transkey1", "testtransactionexec");
    transaction.exec();
    jedis.close();
}`

JedisPool

Use these codes to return a jedis connection to connect redis from the jedis pool.
public static Jedis getJedis(){ JedisPoolConfig config = new JedisPoolConfig(); JedisPool jedisPool = new JedisPool(config, "10.25.19.145", 6379, 60000, "yourpassword"); Jedis resource = jedisPool.getResource(); return resource; }

Using jedispool needs more complicated implementation than connecting to redis directly. However, it provides resources management which is really important in a program of high concurrency.
