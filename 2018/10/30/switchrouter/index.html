Routers perform traffic directing functions on the Internet and forward data packets between computer networks. Switch is a network device that use packet switching to receive process and forward data to the destination device. I will discuss several differences between switches and routers to let you get a better understanding of these concepts. Then, I will discuss the concept of middlebox and the relationship among these three devices.

Differences between Routers and Switches

  • Routers perform at the third layer of the OSI model, which is network layer. However, most switches perform at the second layer of OSI model, which is data link layer. Few switches, which can process data, perform at the third layer.
  • Routers contain routing table which will be used to forward data packet. However, switches will only be used to transfer data traffic without decision making.
  • Routers locate the address by IP and they can process the TCP/IP protocol. However, switches locate the address by MAC.
  • A router can allocate one IP to many devices, while transparent to the outside. However, a switch can connect many hosts, while they all have their own IP.
  • Switches are used to extend Local Area Network (LAN), while Routers can use to connect different LANs, which means that routers can connect different networks.
  • Routers can provide the function of firewalls, while switches cannot.

Middlebox

Middlebox is a computer device that transforms, inspects, filters, or otherwise manipulates traffic for purposes. There are several types of middlebox which are firewalls, intrusion detection system, network address traslator, WAN optimizers, load balancers and so on.

Middlebox is a device which transforms or filters traffic, while it will not forward traffic like a router. In other words, it will not make decisions about which is the next node of the transporting path. However, middle box will make decisions about manipulating the traffic, while switches will only transfer data.
