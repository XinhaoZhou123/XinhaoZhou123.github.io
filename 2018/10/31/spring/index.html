Spring is an open source application framework and inversion of control container for java platform. There are two important features that are really important when we understand spring and program with it. They are Inversion of Control (IoC) and Aspect Oriented Programming (AOP).

Inversion of Control

Inversion of control (IoC) is a design principle, that it is the framework that calls into the custom code. We regard spring as a container which is responsible for managing object life cycles of specific objects: creating these objects, calling their initialization methods, and configuring these objects by writing them together.

Spring is used to decouple the dependencies of these objects. A complex system needs the coroperation of different objects and these objects are coupled in a certain way. By encapsulating these objects and leaving the outside with an interface, the complexity of solving problems is decreased. Spring framework is used to decouple the dependencies.

For example, in java web programming, there are several layers to implement the function. The bean layer initialize the basic object and properties of it. The dao layer perform data persistency by accessing and storing data into database. The service layer perform the transaction and service of the function. The control layer control the implementation of the function chain with the input of the data from customers. As we all know, these four layers are coupled together because they depend on each other to achieve functions. Spring decouples the dependencies between these two functions, injects the dependencies in a form of notation and decreases the complexity of coding.

Aspect Oriented Programming

Aspect-oriented programming (AOP) is a programming paradigm that aims to increase modularity by allowing the separation of cross-cutting concerns. These pointcut is independent in terms of functional logic. However, they are enbeded in the implementaton of these functions and changing one concerns entails the possibility of changing all the concerns. Spring is used to implement the function and perform independency of each step.

For example, in spring, we use AOP configuration to control the transaction. They are used to perform the commit rollback and access the transaction information. For example,when transferring money between accounts, which means increasing the amount of money of one account and decreasing the amount of money of the other, an exception occurs in one account. The rollback of this transaction is performed immediately. In this case, spring performs as the controller of the transaction.
